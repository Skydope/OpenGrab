## Plan — Dependabot + Docker release

### Archivo 1: .github/dependabot.yml

```yaml
version: 2
updates:
  - package-ecosystem: "pip"
    directory: "/"
    schedule:
      interval: "weekly"

  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"

  - package-ecosystem: "docker"
    directory: "/"
    schedule:
      interval: "weekly"
```

### Archivo 2: .github/workflows/docker.yml

```yaml
name: docker

on:
  push:
    tags: ["v*"]

jobs:
  build:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.repository_owner }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/metadata-action@v5
        id: meta
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
      - uses: docker/build-push-action@v6
        with:
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
```

### Cambios vs. plan original

| Cambio | Motivo |
|--------|--------|
| `permissions: packages: write` | Sin esto, GITHUB_TOKEN no pushea a ghcr.io -> 403 |
| Dependabot `github-actions` | Mantiene pineadas las actions (checkout@v4, etc.) |
| Dependabot `docker` | Vigila parches de seguridad de python:3.12-slim |
| Fuera `type=ref,event=branch` | Inerte con trigger solo de tags, no aporta |

### Orden de implementacion

| Paso | Que                                     | Archivo                          |
|------|-----------------------------------------|----------------------------------|
| 1    | Crear .github/dependabot.yml            | .github/dependabot.yml           |
| 2    | Crear .github/workflows/docker.yml      | .github/workflows/docker.yml     |
| 3    | Commit + push                           | —                                |
| 4    | git tag v1.1.0 && git push --tags       | Dispara el build de imagen       |

### Post-deploy manual (una sola vez)

- Entrar a la pagina del package en ghcr.io/skydope/opengrab
- Cambiar visibilidad a "public"
- Vincular al repositorio (Settings > Link to repository)

### Resultado

- Dependabot abre PRs semanales para pip, github-actions y docker base image
- Cada tag v* pushea imagen a ghcr.io/skydope/opengrab con tags semver
- Sin secrets manuales — GITHUB_TOKEN con permissions cubre el registry
- docker/metadata-action normaliza a lowercase, no hay problema con mayusculas
